# Kiro Control Center (kcc) — Design Document

**Date:** 2026-04-02
**Status:** Approved
**Binary name:** `kcc`
**Crate name:** `kiro-control-center`
**App title:** Kiro Control Center

---

## Summary

A lightweight Tauri v2 desktop app for browsing, installing, and managing Claude Code marketplace skills in Kiro projects. Lives alongside the existing `kiro-market` CLI — both share the `kiro-market-core` library crate.

## Motivation

The CLI works well for scripting and single operations, but the browse → select → install workflow is clunky without a visual interface. After adding a marketplace, users have no intuitive way to explore 59 skills across 3 plugins, select what they want, and install in bulk. A GUI solves this with tabs, checkboxes, and one-click install.

## Architecture

### Workspace layout

```
crates/
  kiro-market-core/          # Shared library (unchanged)
  kiro-market/               # CLI binary (unchanged)
  kiro-control-center/       # Tauri app (new)
    src-tauri/
      src/
        main.rs              # Tauri entry point
        lib.rs               # Command registration + specta binding export
        commands/
          mod.rs
          browse.rs          # Marketplace/plugin/skill browsing + install
          installed.rs       # Installed skill listing + removal
          marketplaces.rs    # Marketplace add/remove/update
        error.rs             # CommandError type for Tauri responses
      Cargo.toml
      tauri.conf.json
    src/                     # Svelte 5 frontend
      lib/
        bindings.ts          # Auto-generated by tauri-specta
        components/
          BrowseTab.svelte
          InstalledTab.svelte
          MarketplacesTab.svelte
          SkillCard.svelte
          TabBar.svelte
      routes/
        +layout.svelte       # Tab navigation shell
        +page.svelte          # Main page
      app.html
    package.json
    svelte.config.js
    vite.config.ts
```

### Data flow

```
Svelte Frontend
  │
  │  commands.listAvailableSkills({ marketplace, plugin })
  │
  ▼
Tauri Command Layer (src-tauri/src/commands/)
  │
  │  #[tauri::command] + #[specta::specta]
  │  Resolves CacheDir, calls core functions,
  │  translates errors to CommandError
  │
  ▼
kiro-market-core
  │
  │  cache, plugin, project, skill, git, validation
  │
  ▼
Filesystem (~/.local/share/kiro-market/, .kiro/)
```

### Key design decisions

1. **Tauri commands call kiro-market-core directly** — no subprocess spawning. The core library is a Rust dependency of the Tauri crate, same as the CLI.

2. **tauri-specta generates typed TypeScript bindings** — same pattern as mental-health-bar-rs. Each command is decorated with `#[tauri::command]` and `#[specta::specta]`, and `collect_commands!` generates `bindings.ts` with fully typed request/response types.

3. **No in-memory state cache** — filesystem is the source of truth. Each command reads fresh from disk. This avoids stale state between the app and CLI if both are used.

4. **Project scoped to launch directory** — the app operates on the Kiro project in the current working directory (same as CLI). A project picker can be added later.

5. **CLI preserved unchanged** — `kiro-market` continues to work as before. The Tauri app is a separate binary (`kcc`).

## Tauri Commands

### Browse tab

| Command | Params | Returns | Description |
|---------|--------|---------|-------------|
| `list_marketplaces` | — | `Vec<MarketplaceInfo>` | All registered marketplaces with plugin counts |
| `list_plugins` | `marketplace: String` | `Vec<PluginInfo>` | Plugins in a marketplace with skill counts |
| `list_available_skills` | `marketplace: String, plugin: String` | `Vec<SkillInfo>` | Skills in a plugin with name, description |
| `install_skills` | `marketplace: String, plugin: String, skills: Vec<String>, force: bool` | `InstallResult` | Install selected skills into current project |

### Installed tab

| Command | Params | Returns | Description |
|---------|--------|---------|-------------|
| `list_installed_skills` | — | `Vec<InstalledSkillInfo>` | Skills installed in current project with metadata |
| `remove_skill` | `name: String` | `()` | Remove a skill from the project |

### Marketplaces tab

| Command | Params | Returns | Description |
|---------|--------|---------|-------------|
| `add_marketplace` | `source: String` | `MarketplaceAddResult` | Add marketplace (GitHub shorthand, URL, or path) |
| `remove_marketplace` | `name: String` | `()` | Remove a registered marketplace |
| `update_marketplace` | `name: Option<String>` | `UpdateResult` | Pull latest from remote |

### Utility

| Command | Params | Returns | Description |
|---------|--------|---------|-------------|
| `get_project_info` | — | `ProjectInfo` | Current project path, whether .kiro/ exists |

## Response Types

```rust
// Browse
#[derive(Serialize, specta::Type)]
pub struct MarketplaceInfo {
    pub name: String,
    pub source_type: String,  // "github", "git", "local"
    pub plugin_count: usize,
}

#[derive(Serialize, specta::Type)]
pub struct PluginInfo {
    pub name: String,
    pub description: Option<String>,
    pub skill_count: usize,
    pub source_type: String,
}

#[derive(Serialize, specta::Type)]
pub struct SkillInfo {
    pub name: String,
    pub description: String,
    pub plugin: String,
    pub marketplace: String,
    pub installed: bool,  // Whether already installed in current project
}

#[derive(Serialize, specta::Type)]
pub struct InstallResult {
    pub installed: Vec<String>,
    pub skipped: Vec<String>,
    pub failed: Vec<FailedSkill>,
}

#[derive(Serialize, specta::Type)]
pub struct FailedSkill {
    pub name: String,
    pub error: String,
}

// Installed
#[derive(Serialize, specta::Type)]
pub struct InstalledSkillInfo {
    pub name: String,
    pub marketplace: String,
    pub plugin: String,
    pub version: Option<String>,
    pub installed_at: String,  // ISO 8601
}

// Marketplaces
#[derive(Serialize, specta::Type)]
pub struct MarketplaceAddResult {
    pub name: String,
    pub plugins: Vec<PluginInfo>,
}

#[derive(Serialize, specta::Type)]
pub struct UpdateResult {
    pub updated: Vec<String>,
    pub failed: Vec<FailedUpdate>,
    pub skipped: Vec<String>,
}

#[derive(Serialize, specta::Type)]
pub struct FailedUpdate {
    pub name: String,
    pub error: String,
}

#[derive(Serialize, specta::Type)]
pub struct ProjectInfo {
    pub path: String,
    pub kiro_initialized: bool,
    pub installed_skill_count: usize,
}
```

## Error Handling

Following the mental-health-bar-rs pattern:

```rust
#[derive(Debug, Serialize, specta::Type)]
pub struct CommandError {
    pub message: String,
    pub error_type: ErrorType,
}

#[derive(Debug, Serialize, specta::Type)]
pub enum ErrorType {
    NotFound,
    AlreadyExists,
    Validation,
    GitError,
    IoError,
    Unknown,
}
```

`kiro_market_core::error::Error` is mapped to `CommandError` at the Tauri command boundary. The Svelte frontend can branch on `error_type` for user-friendly messages (e.g., "marketplace not found" vs. "git clone failed").

## UI Design

### Tab layout

```
┌─────────────────────────────────────────────┐
│  [Browse]  [Installed]  [Marketplaces]      │
├─────────────────────────────────────────────┤
│                                             │
│  (tab content)                              │
│                                             │
└─────────────────────────────────────────────┘
```

### Browse tab

- Left sidebar: marketplace → plugin tree navigation
- Main area: skill cards with checkbox selection
- Each card shows: skill name, description, installed badge if already present
- Bottom bar: "Install N selected" button, force-install toggle
- Search/filter input at top

### Installed tab

- List of installed skills sorted by name
- Each row: skill name, plugin@marketplace, version, install date
- Select and "Remove" button
- Search/filter input

### Marketplaces tab

- List of registered marketplaces with source type badge
- "Add marketplace" input field + button at top
- Per-marketplace: Update button, Remove button
- Status indicator (last updated, local/symlinked)

## Scope boundaries

### In v1
- Three tabs: Browse, Installed, Marketplaces
- Bulk skill selection and install
- Individual skill removal
- Marketplace add/remove/update
- Typed TS bindings via tauri-specta
- Project scoped to cwd at launch

### Deferred
- Project picker / scanning for .kiro directories
- Skill preview (reading SKILL.md content)
- Auto-update marketplace on app launch
- Plugin-level install (install all skills in a plugin with one click)
- Drag-and-drop marketplace file import
- Dark/light theme toggle (use system preference for v1)
